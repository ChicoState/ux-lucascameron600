# Microsoft Clipchamp Interaction Journal

For my journal entry I chose to interact with Microsoft Clipchamp. To give some context I am proficcient in other, possibly more proffesional grade, video editing softwares like Sony Vegas Pro. These other editing softwares have much more capability but also a much steeper learning curve. I am hoping that Microsoft Clipchamp will provide a very simple and **familiar** interface, possibly at a trade off for more advanced features as seen in other programs like Sony Vegas Pro.

Upon first loading the program I am greeted with a very **sleek(Aestheticly pleasing)** interface with a home menu and a few large buttons immediatly availible and highlighted. This was exactly what I was hoping this program would provide for me to start. I dont need complicated options and settings I just want something simple. This program is so far doing a good job at being **Learnable.**

In my next step I click on create a new video. At this point it is pretty clear what Id like to do next. There is a large box for me to import my media and instantly start working on it. The tabs on the side inform me I can do other things the media im currently dealing with, but they are not overwhelming. 

<img width="982" height="665" alt="image" src="https://github.com/user-attachments/assets/74a15baa-37f5-44bb-9c41-e20e61044df4" />

I next import my media and start working with it. I quickly added different audio, muted the original videos volume, and changed the aspect ration. While this process did go rather quick, I found that I was constantly frustrated with small bugs in how the program would run. It would cut everything correctly, but it would take a very long time to import new media and watching the video preview it was very choppy. This made it hard for me to edit the video in the exact way that I wanted to, and ended up being very **unsatisfying**. while it didnt truly effect the power of work I was wable to perform, I just didn't love the experience. 

<img width="1895" height="1021" alt="image" src="https://github.com/user-attachments/assets/c784c528-c3e4-408f-886a-57f197953254" />

As mentioned I was quickly able to make the simple edits I wanted, and then I continued to the process of exporting. Exporting was quick and **Efficient**, it just asked what I want to name it and the export aspect ratio. Typically in those other music editing programs, Exporting would be an absolute nightmare of options, because the softwares must suppoort every range of use they end up being extremely complicated. But not this one, the menu was simple sleek and easy to understand.

<img width="857" height="475" alt="image" src="https://github.com/user-attachments/assets/898e3ad4-5ab7-44e3-ac19-f6a217c4a263" />

Overall my experience using microsoft clipchamp was good. I think the desgin was simple, good looking, and easy to use. I was able to quickly do what I wanted to without much limitation, but also without an overwhelming amount of options. The interface was well designed and I did not have to look around **(system status/core functionality visible)** much for what I wanted. If they were able to improve the functionality of the video preview and performance in rendering the preview the experience would become **satisfying** and I would want to use it even more. This program is very simple and usable and while it is not perfect it does a good job at making sure the user has a good expereince.
